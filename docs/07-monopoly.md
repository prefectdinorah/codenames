# Монополия

## Файлы

- **`monopoly-data.js`** — стартовые данные «классической» доски (группы, свойства, транспорт, утилиты, порядок клеток). Колоды `classic` при загрузке из S3 **перезаписываются** эталоном из кода (см. `monopoly-store.js`).
- **`monopoly-store.js`** — загрузка/сохранение JSON **`monopoly/state.json`** в S3, загрузка файлов логотипов под префиксом **`monopoly/logos/`**, кэш в памяти.
- **`public/app.js`** — игровой UI режима.
- **`public/admin.js`** — вкладки колод и логотипов.

## Редактор и доступ

- Список имён с правом админки задаётся в **`server.js`**: `SPECIAL_USER_NAMES` (заголовок **`x-admin-name`** должен совпадать с именем игрока в клиенте для запросов к `/api/admin/*`).
- Публичный эндпоинт **`GET /api/monopoly/decks`** — список id колод для выбора в настройках комнаты.

## S3 layout

- `monopoly/state.json` — объект `{ decks, logos }`.
- `monopoly/logos/<uuid>.<ext>` — бинарники логотипов.

Без ключей S3 всё держится в RAM процесса.

## Игровой поток (высокоуровнево)

Состояние раунда, позиции фишек, торги и т.д. реализованы в **`server.js`** (функции `createMonopolyGame`, `getMonopolyState`, `handleMonopolyMsg`). Детальные типы сообщений — [09-websocket-protocol.md](09-websocket-protocol.md).

## Аукцион, залог и банкротство

- Отказ от покупки свободной клетки запускает открытый аукцион для всех активных игроков.
- Недвижимость/транспорт/ресурсы можно заложить за половину цены; выкуп — залог + 10%.
- Улицу нельзя заложить, пока в её цветовой группе есть постройки: сначала продаются дома/отель.
- Если игроку не хватает наличных для платежа, игра ставит фазу долга: игрок продаёт постройки, закладывает активы или заключает сделки. Банкротство наступает только если наличные + ликвидационная стоимость всех построек и незаложенных владений не покрывают долг.
- Победа наступает, когда остаётся один не-банкрот.

## Реконнект

В обработчике **`join-room`** для `gameMode === 'monopoly'` есть логика **восстановления слота по имени** при отключении — см. комментарии в `server.js`.
