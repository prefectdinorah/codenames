# Обзор проекта

Веб-приложение для **совместных настольных игр в компании** по модели «одна комната — один код из четырёх символов». Игроки открывают сайт с телефона или ноутбука, вводят имя и код комнаты (или создают комнату и выбирают режим).

## Режимы

| Режим (`gameMode`) | Описание |
|--------------------|----------|
| `codenames` | Сетка слов, команды, шпион и оперативники, подсказки и голосование за карточку |
| `alias` | Объяснение слов своей команде за таймер |
| `spyfall` | Один шпион, локация, роли, обвинения |
| `crocodile` | Рисование + угадывание слова командой |
| `whoami` | Слова на лбу / угадывание (назначение слов между игроками) |
| `monopoly` | Упрощённая онлайн-монополия с колодами из редактора |

## Технологии в двух словах

- Один процесс **Node.js**: HTTP + WebSocket на одном порту.
- Состояние игры **в памяти процесса**; персистентность только для **редактора монополии** (и метаданных логотипов) через **S3**, если заданы ключи.

## Связанные файлы в корне

- `server.js` — основная логика.
- `public/` — статический клиент.
- `monopoly-data.js` / `monopoly-store.js` — монополия.
- `Dockerfile`, `docker-compose.yml` — контейнеризация.
